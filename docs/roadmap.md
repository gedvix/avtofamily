# Дорожная карта развития платформы

Документ описывает приоритетные шаги по дальнейшей разработке системы Avtofamily. Этапы можно выполнять параллельно отдельными
командами, но внутри этапа желательно соблюдать порядок, чтобы снизить риски.

## 1. Бэкенд и управление данными
1. **Уточнение модели данных**: добавить справочники для марок/моделей, комплектаций, городов, типов оплат. Настроить миграции и
   фикстуры для быстрого наполнения.  
2. **Пакетные операции**: реализовать импорт/экспорт автомобилей через CSV/Excel, добавить API для массовых обновлений статусов.  
3. **Процесс модерации**: ввести роли пользователей, журнал изменений и правила переходов статусов (черновик → модерация → публикация).  
4. **API для внешних интеграций**: оформить спецификацию (OpenAPI), добавить фильтрацию/сортировку/поиск по каталогу, ввести
   версионирование API.

## 2. Модуль публикаций на сторонние площадки
1. **Единый интерфейс адаптеров**: описать контракт, реализовать демо-адаптер (например, для Telegram) с очередями и ретраями.  
2. **Хранилище учётных данных**: безопасно хранить токены/логины в Secret Manager или Vault, добавить управление из админки.  
3. **Планировщик задач**: настроить Celery + Redis, описать расписания публикаций, обработку ошибок и ручные перезапуски.  
4. **Мониторинг публикаций**: добавить вебхуки/парсеры статусов, страницы аналитики и отчёты для менеджеров.

## 3. Публичный сайт и UX
1. **Фронтенд (Next.js/Nuxt)**: реализовать каталог с фильтрами, карточки авто, лендинг компании, блог. Настроить SSR/SSG и кэширование.  
2. **SEO и аналитика**: добавить Schema.org разметку, OpenGraph, карту сайта, интеграцию с Google Analytics/Tag Manager.  
3. **Формы и коммуникации**: реализовать обратную связь, заявки на тест-драйв, интеграцию с CRM/телефонией.  
4. **Личный кабинет партнёров**: предоставить дилерам/агентам доступ к своим объявлениям, отчётам и статистике.

## 4. Инфраструктура и эксплуатация
1. **CI/CD**: настроить автоматические проверки (pytest, линтеры, Lighthouse), сборку Docker-образов и деплой на staging/production.  
2. **Логирование и мониторинг**: внедрить Sentry, Prometheus/Grafana или ELK, завести алерты в Telegram/Slack.  
3. **Резервное копирование**: регламент бэкапов БД, файлов, проверка восстановления.  
4. **Безопасность**: аудит прав доступа, 2FA, тесты на уязвимости, политика обновления зависимостей.

## 5. План релизов
- **MVP**: базовый каталог, админка с публикацией на Telegram, ручные выгрузки.  
- **Release 1**: интеграции с av.by и Instagram, автоматические отчёты, фильтры по каталогу.  
- **Release 2**: кабинет партнёров, расширенная аналитика, полный набор площадок.  
- **Release 3**: рекомендации, динамическое ценообразование, интеграция с внешними CRM/DMS.

Документ дополняется по мере уточнения требований и результатов исследований площадок.
